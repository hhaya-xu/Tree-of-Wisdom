## 写作工作流

### 日常写作

1. 打开 [[Obsidian]]，在 `content/` 目录下创建/编辑笔记
2. 使用 `[[双向链接]]` 连接相关笔记
3. 保存文件

### 发布到网站

```bash
# 添加所有更改
git add .

# 提交
git commit -m "更新笔记"

# 推送到 GitHub
git push
```

推送后，[[GitHub Actions]] 会自动构建并发布，约 1-2 分钟完成。

### 自动化（可选）

安装 [[Obsidian Git 插件]] 后，每次保存笔记时自动推送到 GitHub。

### 注意事项

- 文件名建议用英文 + 短横线，中文标题写在 frontmatter 的 `title` 字段
- 草稿可加 `draft: true` 在 frontmatter 中，发布时会被过滤
- 私密笔记放在 `content/private/` 目录，会被自动忽略
