## 一、现状诊断

当前 `custom.scss` 有 587 行，但实际上是**两轮独立迭代叠在一起**：

| 轮次 | 位置 | 问题 |
|------|------|------|
| Hallmark Specimen Edition | 文件前半（~L1-L450） | 自己定义了一套完整的色彩/排版/组件样式 |
| Stage 1 CSS Shell | 文件后半（~L450-L587） | 又从头定义了一套 Grid 布局 + 桌面端断点 |

后果：
- `body` 的背景色被定义了两次（一次 `linear-gradient`，一次 `radial-gradient` 叠加）
- Grid 布局在文件末尾用 `!important` 暴力覆盖 Quartz base 的默认 grid
- `.right.sidebar` 的宽度在文件前段是 `340px`，后段又改为 `320px`
- `overflow: hidden` 在 `html`、`body`、`#quartz-root`、`#quartz-body` 上各自重复声明
- 六人物色系统散落在文件后段，与前段的全局配色无关

这不是"改不动"——这是两套方案的残骸互相踩脚。推倒是唯一正确的解法。

---

## 二、重建原则

1. **不放 !important**——用选择器优先级而非暴力覆盖
2. **不覆盖 Quartz base 已有的正确规则**——只覆盖需要改的部分
3. **不重复声明**——每个属性只出现一次
4. **模块化分区**——viewport → background → layout → components → typography → content → mobile
5. **色值统一用 HEX**——方便你在 Obsidian 里肉眼识别

### 4.1 `quartz.config.ts` — 微调配色

只改 `theme.colors.lightMode` 和 `theme.colors.darkMode` 两个区块（内容相同，因为不启用暗色模式）。把以下内容替换到 `quartz.config.ts` 的 `colors` 对象：

```typescript
colors: {
  lightMode: {
    light: "#ede3cf",       // 暖纸底 ≈ bianjing-hours body 渐变起点的实色近似
    lightgray: "#e7dbc4",   // 次表面
    gray: "#c4b898",        // 边框 / 分隔线
    darkgray: "#4a3f35",    // 次级文字
    dark: "#2b2218",        // 正文
    secondary: "#b8860b",   // 链接 / 强调（暗金色）
    tertiary: "#8b6914",    // 链接悬停
    highlight: "rgba(184,134,11,0.08)",  // 内部链接底色
    textHighlight: "#b8860b22",          // 文字高亮
  },
  darkMode: {
    light: "#ede3cf",
    lightgray: "#e7dbc4",
    gray: "#c4b898",
    darkgray: "#4a3f35",
    dark: "#2b2218",
    secondary: "#b8860b",
    tertiary: "#8b6914",
    highlight: "rgba(184,134,11,0.08)",
    textHighlight: "#b8860b22",
  },
},
```

变更点：
- `light`: `#f8f2e8` → `#ede3cf`（更接近 bianjing-hours 的暖金纸色）
- `lightgray`: `#efe8da` → `#e7dbc4`
- `gray`: `#c4b898` → `#c4b898`（不变，这个色温是准的）
- `darkgray`: `#4a3f35` → `#4a3f35`（不变）
- `dark`: `#2b2218` → `#2b2218`（不变）
- `secondary`: `#b8860b` → `#b8860b`（不变，暗金色）
- `tertiary`: `#8b6914` → `#8b6914`（不变）

## 七、如果出问题

1. **构建报错** → 检查 scss 语法（多余分号、括号不匹配）。SCSS 对语法比纯 CSS 严格。
2. **布局异常** → 先看浏览器 DevTools 的 Elements 面板，确认 `.page-header` 的 `height` 是否为 72px。
3. **颜色不对** → 检查 `quartz.config.ts` 的 `light` 是否已改为 `#ede3cf`。`npx quartz build` 会重新从 config 生成 CSS 变量。
4. **中栏不滚** → 检查 `custom.scss` 中的 `overflow-y: auto` 是否正确作用于 `.center`。
