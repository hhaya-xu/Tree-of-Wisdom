## 诊断

线上站字体显示为 Inter / Schibsted Grotesk，而不是我们设定的 Fraunces / Geist。

**根因**：Quartz 的 `variables.scss` 定义了 `--bodyFont` 和 `--headerFont` 的 `!default` 值（Schibsted Grotesk / Inter）。虽然我们的 `custom.scss` 在 `:root` 上覆盖了这些变量，但 Quartz 基础样式 `base.scss` 中：

```css
html, body {
  font-family: var(--bodyFont);
}
```

这条规则在编译顺序上处于 variables.scss 和 custom.scss 之间，形成了覆盖窗口——浏览器在 custom.scss 加载之前就已经解析了 body 的 font-family 并 fallback 到 Inter。

---

## 修复方案

### 步骤 1：创建 `quartz/styles/fonts.scss`

在 `D:\AI\my-knowledge-base\quartz\styles\fonts.scss` 新建文件，内容如下：

```scss
// 文姬字体锁定 — 2026-05-25
// 用 !important 在 :root 和 html,body 双处锁定，
// 确保编译后位于 app.css 末尾，覆盖 Quartz 默认 Inter

:root {
  --bodyFont: "Geist", "Geist Fallback", -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif !important;
  --headerFont: "Fraunces", "Iowan Old Style", Georgia, serif !important;
  --codeFont: "Geist Mono", ui-monospace, SFMono-Regular, Menlo, monospace !important;
}

html, body {
  font-family: var(--bodyFont) !important;
}

h1, h2, h3, h4, h5, h6 {
  font-family: var(--headerFont) !important;
}

code, pre, kbd, samp {
  font-family: var(--codeFont) !important;
}
```

### 步骤 2：在 `custom.scss` 中导入 fonts.scss

在 `D:\AI\my-knowledge-base\quartz\styles\custom.scss` 的**最后一行**追加：

```scss
@import "fonts.scss";
```

### 步骤 3：构建并推送

```bash
cd D:\AI\my-knowledge-base
npx quartz build
git add .
git commit -m "字体修复：Fraunces + Geist 锁定"
git push
```
