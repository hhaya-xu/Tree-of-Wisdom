# 收拾桌子 · 标准操作规范

> v1.0 | 2026-05-26 | 玄策制定
> 适用：玄策 / 明远 / 严律 / 伯喈 / 文姬

---

## 什么是"收拾桌子"

会话结束前，把本次对话中有价值的内容**萃取**出来，写入磁盘。
不靠记忆，靠文件。玄策不能跨对话，只能读磁盘。

## 触发时机

| 触发 | 说明 |
|------|------|
| 会话自然结束 | 默认执行 |
| 主公说"收拾桌子" | 立即执行 |
| 完成重要决策/项目 | 阶段性执行 |
| 对话超过 30 轮 | 主动提议执行 |

## 输出格式

每个角色写入各自指定的路径：

| 角色 | 存储位置 |
|------|---------|
| 玄策 | `会议室/备忘录/YYYY-MM-DD-收拾桌子.md` |
| 伯喈 | `content/系统/知识库经验/` 对应文件 |
| 明远 | `明远-建设备忘录.md` 中追加 |
| 严律 | `严律-建设备忘录.md` 中追加 |
| 文姬 | `文姬-建设备忘录.md` 中追加 |

## 四段式模板

```markdown
# YYYY-MM-DD 收拾桌子

## 本次做了什么
- （列出 3-8 条关键动作，不要流水账）

## 决策
- （本次会话做了哪些选择，为什么）

## 新经验
- ✅ 成功模式：（什么做法有效，可复用）
- ⚠️ 教训：（什么坑踩了，以后怎么避）

## 自检
- [ ] 有没有该登记但漏掉的？（角色/skill/规则/文件）
- [ ] 有没有遗留待办需要移交？
- [ ] 有没有可以写入规则/模板的经验？

## 待办
- [ ] （下次或后续要做的事）
```

## 评判标准

| 等级 | 标准 |
|------|------|
| S | 四段完整 + 自检全部通过 + 经验被写入规则/模板 |
| A | 四段完整 + 自检通过 |
| B | 四段完整，但自检有漏项 |
| C | 缺段或不完整 |
| D | 没做 |

## 与项目日志的关系

| | 项目日志 | 收拾桌子 |
|---|---|---|
| 谁做 | 伯喈 | 各角色自己 |
| 内容 | 纯事实 | 萃取经验 |
| 时机 | 决策后即时 | 会话结束 |

项目日志是"发生了什么"，收拾桌子是"学到了什么"。
收拾桌子可以从项目日志中取材，但不能替代它。
