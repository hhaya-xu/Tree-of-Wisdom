## 一、OMX 执行模式（8 种）—— 待研究

OMX 提供了 8 种执行模式，明远将来可能借鉴：

| 模式 | 含义 | 明远适用？ |
|------|------|----------|
| `autopilot` | 全自动，想法→代码 | 🟡 待评估 |
| `ralph` | 循环执行直到通过，架构师把关 | ⭐ 高优先级 |
| `ultrawork` | 最大并行，多 agent 同时开工 | 🟡 需子代理支持 |
| `team` | N 个 agent 协同处理共享任务 | 🟡 需子代理支持 |
| `pipeline` | 流水线串行协作 | ✅ 已有（策略→执行→审查）|
| `ecomode` | 省钱模式，轻量模型 | 🟡 待评估 |
| `ultrapilot` | 并行自动驾驶+文件归属分区 | 🔴 复杂，Windows 支持弱 |
| `ultraqa` | 测试→验证→修复→循环 | 🟡 需严律配合 |

### 重点：ralph 模式

```
明远执行 → 完成 → 自动触发严律审查 → FAIL → 明远修复 → 再审 → PASS → 报告主公
```

> 核心：明远不自己判断"完成了"，必须经过严律审查。
