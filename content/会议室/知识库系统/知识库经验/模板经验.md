## CSS 壳扩展模板

**模板：** 为新页面类型加样式时，在 `custom.scss` 末尾追加：

```scss
/* [页面类型] */
[data-slug*="路径匹配"] .center article {
  /* 特定规则 */
}
```

所有 Grid/列宽规则包在 `@media (min-width: 1200px)` 内。

**日志条目：** 2026-05-24 · 架构 + 壳 · CSS 壳方案

## 单主题站点锁定

**经验：** 如果站点只有一套视觉方案（如纯暗色），不要在 UI 上留切换开关。在根上锁死：

1. 移除切换按钮（UI 层）
2. `lightMode = darkMode`（配置层，HEX 值以兼容 OG 图片生成）
3. 强制脚本：`saved-theme="dark"` + `nav` 事件兜底（运行时层）

**适用场景：** Quartz、或其他有主题切换功能的 SSG。

**日志条目：** 2026-05-24 · 暗色锁定

## 框架页协作工作流

**模板：** 用户调整样式时，不走"描述→代码→部署→反馈"长链路：

1. 从 `public/` 生成独立 HTML，CSS 全量内嵌
2. 用户直接改 `<style>` 标签，浏览器实时预览
3. 满意后伯喈对比差异，同步到 `custom.scss` / `topbar.scss` / `quartz.layout.ts`
4. `npx quartz build` → `git push` → GitHub Actions 部署

**适用场景：** Quartz 样式迭代；任何 SSG + 非技术用户协作场景。

**日志条目：** 2026-05-24 · 布局重构

## CSS 覆盖排查清单

**模板：** Quartz custom.scss 不生效时，按以下顺序排查：

1. 浏览器默认值（`body{margin:8px}`）
2. base.scss 高具体度规则（`.page > #quartz-body .sidebar` = 0,2,1）
3. Grid 残留（`grid-template-areas` / `column-gap` / `row-gap`）
4. 组件 scss 加载顺序（`footer.scss` 在 `custom.scss` 之后）
5. 子选择器 `>` 陷阱（DOM 层级与 CSS 选择器不匹配）

**口诀：** 从外到内，从低到高，先查加载顺序再查具体度。

**日志条目：** 2026-05-24 · 布局重构（4 层阻隔逐一击破）
